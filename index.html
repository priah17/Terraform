Virtualization Unveiled

A concise educational guide to KVM, Docker, and Kubernetes — what they are, how they differ, and quick hands-on examples.

Introduction

Virtualization and containerization are foundational technologies for modern infrastructure. This page explains three major technologies at different layers:

  • KVM — hardware virtualization (full VMs).
  • Docker — container runtime (process isolation using OS features).
  • Kubernetes — orchestration system for running containers at scale.

KVM — Kernel-based Virtual Machine

KVM is a Linux kernel module that turns the kernel into a hypervisor. It enables full virtual machines with separate kernels. Each VM runs a complete OS and has strong isolation and dedicated virtualized hardware.

1
Use cases: multi-tenant isolation, running different OS versions, legacy apps.
2
Pros: strong isolation, hardware acceleration (VT-x, AMD-V).
3
Cons: higher overhead — memory and boot time compared to containers.

Quick Commands

# install (Debian/Ubuntu)
sudo apt update
sudo apt install qemu-kvm libvirt-daemon-system libvirt-clients virtinst

# create a VM from ISO (example)
virt-install --name testvm --vcpus 2 --memory 2048 --cdrom /path/to/ubuntu.iso --disk size=20

# list VMs
virsh list --all

Docker — Containers & Image-based Packaging

Docker packages applications and their dependencies into images and runs them as lightweight containers using namespaces and cgroups for isolation. Containers share the host kernel, making them fast and resource-efficient.

Use
Microservices, reproducible builds, developer workflows.
Pros
Low overhead, fast startup, portable images.
Cons
Weaker kernel-level isolation than VMs; kernel compatibility matters.

Dockerfile Example

FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt ./
RUN pip install -r requirements.txt
COPY . ./
CMD ["python","app.py"]

Quick Commands

# build image
docker build -t myapp:1.0 .

# run container
docker run --rm -p 8000:8000 myapp:1.0

# list running containers
docker ps

Kubernetes — Orchestration at Scale

Kubernetes (k8s) schedules and manages containerized applications across a cluster of machines. It provides service discovery, load balancing, self-healing, scaling, and declarative configuration via YAML manifests.

Control
Declarative control plane: API server, etcd, scheduler, controller-manager.
Node
Worker nodes run kubelet and container runtime (CRI).
Objects
Pods, Services, Deployments, StatefulSets, DaemonSets.

Sample Deployment (YAML)

apiVersion: apps/v1
kind: Deployment
metadata:
  name: myapp-deploy
spec:
  replicas: 3
  selector:
    matchLabels:
      app: myapp
  template:
    metadata:
      labels:
        app: myapp
    spec:
      containers:
      - name: myapp
        image: myorg/myapp:1.0
        ports:
        - containerPort: 8000

Quick Commands

# apply manifest
kubectl apply -f deployment.yaml

# view pods
kubectl get pods

# scale
kubectl scale deployment myapp-deploy --replicas=5

Side-by-side Comparison

KVM

Full VMs, hardware-level virtualization, heavier but stronger isolation. Good for running different OSes and multi-tenant workloads.

Docker

Lightweight containers sharing host kernel. Fast startup and perfect for microservices and dev workflows.

Kubernetes

Orchestrator to run containers across clusters. Handles scaling, healing and service discovery — not a runtime itself.

When to choose what

  • Need multiple OS kernels? KVM.
  • Need fast, reproducible app packaging? Docker.
  • Need to run many containers across machines with auto-scaling? Kubernetes.

Glossary & Resources

KVM
Kernel-based Virtual Machine — an in-kernel hypervisor for Linux.
Namespace
Linux kernel feature isolating process view of resources (PID, network, mount).
Cgroup
Control groups — kernel feature to limit resource usage (CPU, memory, IO).
Pod
Smallest deployable unit in Kubernetes — one or more containers that share network and storage.
